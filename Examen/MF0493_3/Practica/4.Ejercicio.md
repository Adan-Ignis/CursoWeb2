4. Una aplicación necesita almacenar grandes volúmenes de datos.
¿Qué tipo de sistema utilizarías y qué aspectos tendrías en cuenta?

1 - Tipo de sistema recomendado

Dependiendo de la estructura de los datos, las dos opciones principales son:

    Data Lake (Lago de Datos): Si los datos son no estructurados o crudos (imágenes, vídeos, logs, documentos). Se suelen utilizar sistemas de almacenamiento de objetos como Amazon S3, Azure Blob Storage o Google Cloud Storage.

    Data Warehouse (Almacén de Datos): Si los datos son estructurados y el objetivo es realizar análisis de negocio complejos. Ejemplos: BigQuery, Snowflake o Amazon Redshift.

    Sistemas de Archivos Distribuidos: Para procesamiento masivo paralelo, como HDFS (Hadoop Distributed File System).

2 - Aspectos clave a tener en cuenta

Al diseñar o seleccionar la infraestructura, debes evaluar los siguientes pilares:

A. Escalabilidad
    Escalabilidad Horizontal (Scaling out): Capacidad de añadir más nodos al clúster en lugar de hacer más potente un solo servidor. Es vital para manejar el crecimiento indefinido.

B. Rendimiento y Latencia
    Velocidad de escritura/lectura (IOPS): ¿La aplicación necesita guardar datos en tiempo real o por lotes (batch)?

    Indexación: Para grandes volúmenes, una buena estrategia de particionado e indexación es la diferencia entre una consulta de milisegundos y una de varios minutos.

C. Disponibilidad y Tolerancia a Fallos
    Replicación: Los datos deben estar duplicados en diferentes zonas geográficas o servidores para evitar pérdidas si un hardware falla.

    SLA (Service Level Agreement): Asegurar que el sistema esté activo el 99.9% del tiempo.

D. Coste (TCO - Total Cost of Ownership)
    Almacenamiento en frío (Cold Storage): Para datos que rara vez se consultan pero deben conservarse (es mucho más barato).

    Almacenamiento en caliente (Hot Storage): Para datos de acceso frecuente (más caro, pero más rápido).

E. Seguridad y Cumplimiento
    Cifrado: Tanto en reposo como en tránsito.

    Normativas: Asegurarse de cumplir con leyes como el RGPD (Reglamento General de Protección de Datos) si se almacenan datos personales.
