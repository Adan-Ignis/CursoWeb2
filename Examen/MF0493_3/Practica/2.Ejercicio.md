2. Tienes una aplicación web que será accesible desde internet.
Describe los pasos básicos para desplegarla en un servidor Linux.

1 - Prepración del Entorno (Acceso y Actualización)

Antes de instalar nada, es vital conectarse al servidor y asegurarse de que el sistema esté al día para evitar vulnerabilidades de seguridad.

    Acceso: Conéctate vía SSH (Secure Shell) usando el comando:

        ssh usuario@direccion_ip
    
    Actualización: Ejecuta la actualización de los repositorios y paquetes del sistema

2 - Instalación de Dependencias (Runtime)

Debes instalar el software necesario para que tu aplicación se ejecute. Esto varía según el lenguaje:

    Instala los intérpretes o entornos de ejecución correspondientes. Por ejemplo: Node.js, Python, PHP y/o Ruby

    Instala gestores de paquetes como para descargar las bibliotecas que usa tu web. Por ejemplo: npm, pip o composer

3 - Configuración de la Base de Datos

Si tu aplicación guarda información, necesitarás un motor de base de datos

    Instala el servidor. Por ejemplo: MySQL, PostgreSQL o MongoDB

    Crea la base de datos, el usuario y otorga los permisos necesarios.

    Asegúrate de que la base datos solo escuche perticiones locales (127.0.0.1) por seguridad, a menos que el servidor de DB sea externo.

4 - Despliegue de Código y Servidor Web (Reverse Proxy)

El servidor web actúa como el "anfritrión" que recibe las visitas de internet.

    Clonación: Descarga tu código desde un repositorio (usualmente con "git clone")

    Servidor Web: Aunque las aplicaciones modernas suelen tener su propio servidor (como Gunicorn para Python o PM2 para Node), se recomienda usar "Nginx" como proxy inverso.

        Nginx recibe la petición en el puerto 80/443 y la redirige internamente al puerto donde corre tu app (ej. 3000)

5 - Gestión de Procesos (Mantenimiento Vivo)

Si cierras la terminal de SSH, tu aplicación podría detenerse. Para evitarlo:

    Usa un gestor de procesos como systemd o PM2. Esto garantiza que la aplicación se reinicie automáticamente si el servidor se apaga o si el software falla.

6 - Seguridad y Firewall

Este es el paso crítica para que la web sea accesible pero segura:

    Firewall (UFW): Configura el firewall para que solo permita tráfico en los puertos necesarios:

        22 (SSH)

        80 (HTTP)

        443 (HTTPS)
    
    Certificado SSL: Instala un certificado (puede usar Let's Encrypt con la herramienta Certbot) para que la comunicación esté cifrada y aparezca el candado de "Sitio seguro".

7 - Configuración de DNS

Finalmente, debes ir al panel de control de tu dominio y crear un Registro A que apunte la URL (ej: www.tuweb.com) a la dirección IP pública de tu servidor Linux.
